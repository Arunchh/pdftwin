## What is translated (Phase 1 — shipped)

- Home hero, tool grid, trust bar, formats section
- Site navigation, footer, pricing page (plans + key FAQ)
- Tool names and descriptions in nav/grid
- SEO: `<html lang>`, canonical URLs, `hreflang` alternates
- **Locale SEO landing pages** for long-tail search (see below)
- **Language switcher** in the main navigation bar (desktop + mobile menu)

**Not yet translated:** in-tool panel UI (merge workflow, convert forms, etc.), login/account/privacy/terms pages.

### Implementation

Content lives in [`frontend/src/i18n/seoLandings.ts`](../../frontend/src/i18n/seoLandings.ts). Routes:

- English: `frontend/src/pages/guides/[slug].astro`
- Localized: `frontend/src/pages/[locale]/[slug].astro`

Each landing page includes:

- Unique `<title>` and meta description per locale
- Cross-locale `hreflang` links via `seoHreflangAlternates()` — mapped by `toolId`, not slug
- FAQ structured data (`FAQPage` schema) when FAQ items exist
- CTA to the corresponding `/tools/` workspace (locale-aware via `toolPath()`)
- Related guide links (`relatedSlugs` — must reference slugs in the **same locale**)

Sitemap: all 48 URLs listed in [`frontend/public/sitemap.xml`](../../frontend/public/sitemap.xml).

### Adding a new SEO landing

1. Add a `SeoLanding` entry to **each** locale array in `seoLandings.ts` (EN, ES, FR, NL)
2. Set `toolId`, localized slug, copy, and `relatedSlugs` (same-locale slugs only)
3. Add all four URLs to `frontend/public/sitemap.xml`
4. Run `npm run build` in `frontend/` — verify new paths appear under `guides/[slug]` and `[locale]/[slug]`

## Auto language detection

On Vercel, [`middleware.ts`](../../middleware.ts) runs at the edge:

1. **Skip** if URL already has `/es/`, `/fr/`, or `/nl/`
2. **Skip** if `pdftwin_locale` cookie is set (manual choice)
3. **Skip** English `/guides/*` paths (or redirect to localized SEO slug when geo/lang detects non-English)
4. **Detect** using:
   - `x-vercel-ip-country` (e.g. ES → Spanish, NL → Dutch, FR → French)
   - `Accept-Language` header (browser preference)
   - Belgium (`BE`) uses Accept-Language to choose French vs Dutch
5. **Redirect** to localized path (307), preserving query string

Manual override: **language switcher** in the nav bar sets cookie + navigates. On SEO landing pages, switching locale maps to the equivalent localized slug (e.g. `/guides/merge-pdf-free` → `/es/unir-pdf`) via `resolveLocaleSwitchPath()`.

| [`frontend/src/components/seo/ToolSeoLanding.tsx`](../../frontend/src/components/seo/ToolSeoLanding.tsx) | Landing page UI |

### Adding a string

1. Add key to `frontend/src/i18n/types.ts` (`Messages` interface)
2. Add English copy in `locales/en.ts`
3. Add Spanish, French, Dutch in `es.ts`, `fr.ts`, `nl.ts`
4. Use `const { messages } = useI18n()` in React components

### Adding a localized page

1. Create English page under `src/pages/`
2. Mirror under `src/pages/[locale]/` with `getStaticPaths()` returning `NON_DEFAULT_LOCALES`
3. Pass `locale` to `BaseLayout`

## SEO notes

- Each locale has unique URLs (`/es/pricing`, not query params)
- `hreflang` links on every localized layout
- SEO landings use tool-based hreflang mapping (slugs differ per language)
- `x-default` points to English
- All landing URLs are listed in `sitemap.xml`
